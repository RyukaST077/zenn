## 上位候補の詳細

### 1. Zodしか知らない新人がArkTypeを型主導構文で書き比べてみた

#### 概要
Zodの `z.object({...})` に慣れた新人が、ArkType 2.2 の型主導構文（`type({...})`）で同じバリデーションを書き、型推論・エラー表示・書き味の違いをまとめる。筆者の「書き比べてみた」シリーズ（Temporal vs Date, using vs try/finally, Map.getOrInsert 等）の型安全ライブラリ版。

#### なぜ今よさそうか
- ArkType は 2026年の fastest-growing npm パッケージの常連で、Zod代替の筆頭。TSKaigi 2026 でも Standard Schema 文脈で言及。
- 2.2.3 が直近リリース（2026/7時点で数日前）で情報が新しい。

#### 新人が検証しやすい理由
`npm i arktype` だけで開始でき、追加の環境・認証・課金が一切不要。tsc と node の実行結果（型エラー・バリデーション結果）だけで記事が成立する。

#### 想定読者
Zodは使ったことがあるが、Valibot/ArkTypeは名前しか知らない新人〜2年目。

#### タイトル案
1. Zodしか知らない新人がArkTypeを触って書き比べてみた
2. ArkTypeの型主導構文をZodと同じschemaで書いてみた
3. Zod → ArkType、最初に詰まった型の書き方の違い

#### 検証でやること
- 同じ入力バリデーション（ネストobject / union / optional / regex）を Zod と ArkType で書く
- 型推論の効き方（`infer` 相当）と、不正データ投入時のエラーメッセージを比較
- ArkType 2.2 の `type.fn`（ランタイム検証付き関数）を軽く試す

#### 検証手順
1. `npm init` → `npm i arktype zod typescript` でプロジェクト作成
2. 同一の「ユーザー登録フォーム」schema を両方で記述
3. 正常系・異常系データを流し、返るエラー構造を比較
4. `tsc --noEmit` で型推論結果（ホバー型）を確認・スクショ
5. ArkType特有の構文（文字列DSL）で詰まった点を記録

#### 記事構成案
1. はじめに
2. なぜArkTypeを試すのか
3. 事前に調べたこと（Standard Schema / fastest-growing）
4. 環境構築
5. 同じschemaをZodとArkTypeで書く
6. 詰まった点（文字列DSL・型エラーの読み方）
7. 触ってみて分かったこと
8. Zodと比べて感じたこと
9. どんな人に向いていそうか
10. まとめ

#### 差別化ポイント
過去記事は「Node/JS標準機能の書き比べ」が中心で、型安全ライブラリの比較は未着手。新人目線の「Zodからの移行で最初に詰まる所」に絞ると既存の網羅比較記事と差別化できる。

#### 参考リンク
- https://arktype.io/docs/blog/2.2
- https://www.npmjs.com/package/arktype
- https://www.pkgpulse.com/guides/valibot-vs-arktype-vs-zod-2026

#### 注意点
網羅的な性能比較にすると「専門家の解説」になりがち。あくまで「新人が同じschemaを書いて詰まった記録」に徹する。

### 2. CSS contrast-color() で文字色の白黒切替をJSなしで試してみた

#### 概要
背景色を渡すと white / black のうちコントラストが高い方を返す `contrast-color()` を使い、ボタンやバッジの文字色をJSなしで自動切替する。筆者のCSS-tryシリーズ（Anchor Positioning）の系譜で、Playwrightでスクショ確認する。

#### なぜ今よさそうか
2026年4月に Baseline Newly available になったばかりの新機能で、「デザイントークンから文字色を自動決定」という実務ニーズに直結する。

#### 新人が検証しやすい理由
HTML+CSSファイル1枚で完結。Playwright で localhost を開いてスクショを撮るだけで検証・記事素材が揃う。

#### 想定読者
アクセシビリティ（コントラスト比）に興味のある新人〜2年目。

#### タイトル案
1. CSS contrast-color() で文字色の白黒をJSなしで自動切替してみた
2. Baseline入りした contrast-color() を新人が試してみた
3. contrast-color() で中間色が読めなくて詰まった話

#### 検証でやること
- 複数の背景色に対し `color: contrast-color(var(--bg))` を適用しスクショ
- 中間色（例 #2277d3）で読みにくくなる制約を再現・確認
- Playwright のChromiumでの対応状況を確認（過去のbrowser-gate失敗を踏まえバージョン確認を最初に）

#### 検証手順
1. 背景色を並べたHTML/CSSを作成
2. `npx playwright` でスクショ取得
3. Chromiumバージョンと対応可否を最初に確認（未対応なら記録して切替）
4. 中間色の読みにくさをコントラスト比とともに記録
5. `color-mix()` との併用も試す

#### 記事構成案
1. はじめに
2. なぜcontrast-color()を試すのか
3. 事前に調べたこと（Baseline / WCAG）
4. 環境構築（Playwright）
5. 実際に試したこと
6. 詰まった点（中間色・ブラウザ対応）
7. 触ってみて分かったこと
8. 既存手法（複数色定義）と比べて感じたこと
9. どんな人に向いていそうか
10. まとめ

#### 差別化ポイント
既存記事の多くは「使い方紹介」止まり。中間色で読めなくなる制約をコントラスト比つきで実測し、新人目線の落とし穴として書くと差がつく。

#### 参考リンク
- https://developer.mozilla.org/en-US/docs/Web/CSS/Reference/Values/color_value/contrast-color
- https://web.dev/baseline/2026
- https://piccalil.li/blog/some-css-only-contrast-options-until-contrast-color-is-baseline-widely-available/

#### 注意点
過去にChrome版のproperty gate/起動で止まった記録あり。Playwright同梱Chromiumのバージョンを最初に確認し、未対応時は「止まった記録」として成立させる方針を持っておく。

### 3. 「Valibotはバンドル90%削減」は本当か、Zodと同じschemaで実測してみた

#### 概要
「Valibot は Zod 比で90%小さい」という定番の主張を、同じログインフォームschemaを両方で書いてバンドルサイズを実測して確かめる。筆者のベンチマーク実測系（Vite Rolldown 8倍→実際1.9倍）の型検証版。

#### なぜ今よさそうか
Valibot は fastest-growing の常連で、edge/browser 向けのバンドル削減が最大の訴求。数字が独り歩きしがちなので実測記事の価値が高い。

#### 新人が検証しやすい理由
`npm i` とバンドラ（esbuild/rollup）でサイズを出すだけ。認証・課金・デプロイ不要でAI単独完結。

#### 想定読者
バンドルサイズを気にし始めた新人〜2年目、フロント/エッジ開発者。

#### タイトル案
1. 「Valibotは90%小さい」を新人が同じschemaで実測してみた
2. Zod vs Valibot、ログインフォームschemaのバンドルを測ってみた
3. Valibotのtree-shakingで詰まった点と実測値

#### 検証でやること
- 同一schema（login form）を Zod / Zod Mini / Valibot で記述
- esbuild で minify+gzip サイズを実測し表にまとめる
- 記事の主張値（1.37KB vs 17.7KB 等）と自分の実測を突き合わせる

#### 検証手順
1. `npm i zod valibot esbuild`
2. 3パターンの入口ファイルを用意
3. `esbuild --bundle --minify` でサイズ計測、gzip後も記録
4. tree-shaking が効く書き方/効かない書き方を比較
5. 実測値と公称値の差を考察

#### 記事構成案
1. はじめに
2. なぜバンドルサイズを測るのか
3. 事前に調べたこと（公称値）
4. 環境構築
5. 実測（表つき）
6. 詰まった点（import方法でサイズが変わる）
7. 分かったこと
8. Zodと比べて感じたこと
9. どんな人に向いていそうか
10. まとめ

#### 差別化ポイント
「公称値をうのみにせず自分で測る」姿勢は既存の比較記事と差別化しやすく、筆者の実測系シリーズと一貫する。

#### 参考リンク
- https://www.pkgpulse.com/guides/valibot-vs-zod-v4-typescript-validator-2026
- https://www.npmjs.com/package/valibot

#### 注意点
測定条件（minify/gzip/import方式）を明記しないと数字が誤解される。条件を必ず書く。
