## 上位候補の詳細

### 1. スクロール完了をPromiseで待つ（Programmatic Scroll Promises）を旧来手法と書き比べ

#### 概要

Chrome 150で `window.scrollTo()` / `element.scrollBy()` / `element.scrollIntoView()` がスクロールアニメーション完了時にresolveするPromiseを返すようになった。これまで `scrollend` イベントや `setTimeout` の当て推量で待っていた処理を `await` で書ける。あわせて、`scrollIntoView()` は完了前にresolveしてしまう（2026年7月時点の既知挙動）というズレを実測する。

#### なぜ今よさそうか

- Chrome 150（2026年6〜7月）で来たばかりの新API。国内の「実際に測ってみた」記事はまだ少ない。
- 「便利になった」で終わらせず、**公式APIと実挙動のズレ**という一次検証の価値がある（Chromium issue #41406914 でも議論中）。

#### 新人が検証しやすい理由

- 数十行のHTML/JSで完結。ビルド不要。
- Playwright（Chromium）で `await scrollTo` の解決タイミング・スクロール位置・スクショを機械的に取れる。人手不要。

#### 想定読者

新人〜実務2年目のフロントエンド。scrollendやsetTimeoutで「スクロール後に処理」を書いた経験がある層。

#### タイトル案

1. スクロール完了を `await` で待てるようになったので、setTimeout / scrollend と書き比べてみた
2. Chrome 150 のスクロールPromiseを試したら、scrollIntoView だけ完了前に resolve した記録
3. 新人が「スクロール終わったら実行」を3つの書き方で比べてみた

#### 検証でやること

- `scrollTo` / `scrollBy` / `scrollIntoView` それぞれのPromise解決を計測
- 旧来手法（`scrollend` リスナ / `setTimeout(_, 500)`）と、コード量・正確さ・タイミングを比較
- `scrollIntoView` が完了前にresolveするかをスクロール位置ログで確認

#### 検証手順

1. 縦長のHTML（アンカー付き）を用意し、`node`/静的サーバで配信
2. Playwrightで各scrollメソッドを `await` し、解決時刻とその瞬間の `scrollY` を記録
3. `scrollend` / `setTimeout` 版を同条件で実装し同じ計測を取る
4. 各手法のスクショと計測値を表にまとめる
5. `scrollIntoView` の early-resolve を再現できるか確認し、できれば回避策（`scroll`+待機）も試す

### 2. CSS `text-fit` で見出しを箱幅にフィットさせ、JS/SVGフィットと比べてみた

#### 概要

Chrome 150の `text-fit` プロパティで、テキストのフォントサイズを親要素の幅に自動フィットできる。従来はJS（fitty等）やSVG `textLength` ハックが必要だった処理を、CSSだけで置き換えられるかを検証する。

#### なぜ今よさそうか

- 出たばかりで国内記事がほぼ無い視覚系CSS。ビフォーアフターのスクショが映える。
- 既存記事「Gap Decorations（失敗）」「focusgroup（gate停止）」と違い、**成功して視覚結果を見せられる**CSSネタ。

#### 想定読者

## 最初に試すべき1本

### 推奨テーマ

**スクロール完了をPromiseで待つ（await scroll）を setTimeout / scrollend と書き比べ、resolveタイミングのズレまで測る**（候補#1）

### 理由

- **トレンド性**: Chrome 150で来たばかり（2026年6〜7月）の新DOM API。
- **新人でも検証できる**: 数十行のHTMLとPlaywrightだけ。ビルド・課金・サインアップ不要でAIエージェント単独完結。
- **Zenn読者に刺さる**: 「スクロール後に処理を走らせる」は誰もがsetTimeoutで妥協してきた定番の悩み。置換提案は実用的。
- **半日〜1日で検証できる**: 3手法の計測と比較表・スクショで十分1本になる。
- **完成イメージ**: 「3つの書き方の計測表 + scrollIntoViewだけ完了前にresolveした観測ログ + 回避策」。紹介記事に埋もれない一次検証になる。

### 最小検証プラン

1. アンカー付きの縦長HTMLを用意し、ローカル配信
2. Playwright(Chromium)で `await scrollTo/scrollBy/scrollIntoView` の解決時刻と `scrollY` を記録
3. `scrollend` 版・`setTimeout` 版を同条件で実装して計測
4. 3手法をコード量/正確さ/タイミングで表にまとめ、`scrollIntoView` のearly-resolveを検証
5. 観測結果・スクショ・回避策を execution-log に残す
