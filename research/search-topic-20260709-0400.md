## 上位候補の詳細

### 1. ESLintしか知らない新人がoxlintに移行し、JSプラグインalphaまで試してみた

#### 概要
Rust製リンタ oxlint に既存 ESLint プロジェクトから移行し、`@oxlint/migrate` で設定を変換、速度差を実測し、2026-03にalphaへ到達した「JSプラグイン」機能（既存ESLintプラグインをそのまま動かす）まで新人目線で試す検証ログ。

#### なぜ今よさそうか
公式ブログ（2026-03-11）でJSプラグインalphaが公表され、oxlintが「ESLintのドロップイン置換」に現実的に近づいた直近の一次情報がある。50〜100倍高速という話題性も強い。

#### 新人が検証しやすい理由
`pnpm add -D oxlint` → `.oxlintrc.json` → `pnpm run lint` の3ステップで動く。認証・課金・クラウド不要でローカル完結。

#### 想定読者
ESLintは使っているがoxlintは未経験の新人〜2年目。移行できるか・何で詰まるかを知りたい層。

#### タイトル案
1. ESLintしか知らない新人がoxlintに移行してみた（JSプラグインalphaまで）
2. oxlintへの移行で詰まったところと、速度を実測した話
3. 話題のoxlintを1日試して、ESLintプラグインが本当に動くか確かめた

#### 検証でやること
- 既存ESLint設定のある小さなReact/TSプロジェクトを用意
- `@oxlint/migrate` で `.oxlintrc.json` を生成し差分を確認
- ESLint と oxlint の lint 実行時間を同じ対象で計測
- `jsPlugins` に既存ESLintプラグインを指定して動くか検証（alphaの制約も記録）

#### 検証手順
1. サンプルプロジェクト作成＋ESLint flat config を用意
2. `pnpm add -D oxlint` と migrate ツールで設定変換
3. `oxlint` を実行、ESLint と実行時間・検出差を比較
4. `jsPlugins` に `eslint-plugin-*` を1つ足して挙動と制限を確認
5. 詰まった点（未対応ルール・alpha由来のエラー）をログ化

#### 記事構成案
1. はじめに 2. なぜ今oxlintを試すのか 3. 事前に調べたこと（alphaの位置づけ） 4. 環境構築 5. 移行と実測 6. JSプラグインを試す 7. 詰まった点 8. 分かったこと 9. ESLintと比べて 10. どんな人向けか 11. まとめ

#### 差別化ポイント
既存のoxlint紹介記事は多いが、「新人が移行し、2026-03のJSプラグインalphaまで自分で試して詰まる」実践ログは希少。速度実測＋alpha制約の一次記録で差別化。

#### 参考リンク
- oxc.rs blog: Oxlint JS Plugins Alpha (2026-03-11)
- oxc.rs docs: Migrate from ESLint / @oxlint/migrate
- LogRocket / Better Stack: oxlint vs ESLint 2026

#### 注意点
JSプラグインは**alpha**。未対応・不安定を前提に、うまくいかない点も価値として記録する。速度比較は環境・対象規模を明記。

### 2. Zodしか知らない新人がArkTypeを触り、Standard Schemaで差し替えてみた

#### 概要
TypeScript構文そのままでスキーマを書く ArkType を、Zod経験しかない新人が試す。Standard Schema v1.0 により Zod/ArkType が相互運用できる点を利用し、同じ検証ロジックを差し替えて型・実行時検証・速度感の違いを確かめる。

#### なぜ今よさそうか
Standard Schema v1.0 の普及で検証ライブラリの「差し替え」が現実的になり、ArkTypeはZod比3〜4倍速という話題性がある。比較記事需要が国内外で高い。

#### 新人が検証しやすい理由
ローカルにnpmで入れて小さなスキーマを書くだけ。認証・課金・サーバー不要。1〜2時間で最小検証可能。

#### 想定読者
Zodは使えるが他の検証ライブラリは未経験の新人〜2年目。

#### タイトル案
1. Zodしか知らない新人がArkTypeを触ってみた
2. Standard Schemaで検証ライブラリを差し替えたら何が変わるか試した
3. ArkTypeの「TypeScriptそのまま」なスキーマを新人目線で整理してみた

#### 検証でやること
- 同じ入力バリデーションをZodとArkTypeで書き比べ
- 型推論・エラーメッセージ・safeParse相当の使い勝手を比較
- Standard Schema対応の受け口で両者を差し替えられるか確認

#### 検証手順
1. TS+tsx でローカル検証環境を用意
2. Zodでスキーマを書く 3. 同じものをArkTypeで書く
4. 不正データで検証エラーの出方を比較
5. Standard Schema経由で差し替え、詰まった点を記録

#### 記事構成案
1. はじめに 2. なぜArkTypeを試すのか 3. 事前に調べたこと 4. 環境構築 5. 書き比べ 6. 詰まった点 7. 分かったこと 8. Zodと比べて 9. どんな人向けか 10. まとめ

#### 差別化ポイント
「新人がZodの知識だけでArkTypeに触れて詰まる」実践ログ＋Standard Schema差し替えの実験。比較の紹介記事ではなく手を動かした一次記録。

#### 参考リンク
- arktype.io docs / Integrations
- pkgpulse: Zod v4 vs ArkType vs Valibot 2026
- Zenn(en) m_noto: Zod代替の比較

#### 注意点
型レベルの挙動は難しい部分があるので「新人が理解できた範囲」を明示。速度比較は簡易ベンチである旨を注記。

## 最初に試すべき1本

### 推奨テーマ
**ESLintしか知らない新人がoxlintに移行し、JSプラグインalphaまで試してみた**（候補#1 / 合計28点）

### 理由
- **トレンド性**: 2026-03-11の公式一次情報（JSプラグインalpha）があり直近性が高い。Rust製ツールチェーンの注目波にも乗る。
- **新人でも検証できる**: 3ステップで動作。認証・課金・クラウド不要でローカル完結し、AIエージェント単独で最後まで実行できる。
- **Zenn読者に刺さる**: ESLint移行は多くの現場の関心事。速度実測＋alpha機能の詰まり所は保存性が高い。
- **半日〜1日で検証可能**: 最小構成なら1〜2時間で移行と実測まで到達できる。
- **完成イメージ**: 「移行手順→実測グラフ的な数値→JSプラグインで動いた/動かなかった→詰まった点」の検証ログ記事。

### 最小検証プラン
1. ESLint flat config を持つ小さなReact/TSサンプルを作る
2. `pnpm add -D oxlint` ＋ `@oxlint/migrate` で `.oxlintrc.json` を生成
3. `oxlint` 実行、ESLintと実行時間・検出差を計測してログ化
4. `jsPlugins` に既存ESLintプラグインを1つ指定して挙動を確認
5. 詰まった点・alphaの制約をスクショ/ログとともに記録
