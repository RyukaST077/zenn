## 上位候補の詳細

### 1. `randomUUIDv7()` と `randomUUID()`(v4) を書き比べ、SQLiteの並び順で違いを見る

#### 概要
Node 26.1 で標準に入った `crypto.randomUUIDv7()` と、従来の `crypto.randomUUID()`（v4）を同じコードで並べて生成し、(1) 文字列として辞書順ソートしたときの並び、(2) 先頭48bitからのタイムスタンプ復元、(3) SQLiteに主キーとして連続INSERTしたときの並び順/生成順一致を観察する。ライブラリ不要・完全ローカル。

#### なぜ今よさそうか
- 「UUIDv4をやめてv7へ」という話題が国内外で強い一方、日本語で**Node標準API単体を手を動かして確かめた記事**はまだ薄い。
- v7の「利点」は語られがちだが、実際に生成して並びやtimestampを見た一次ログは価値が高い。

#### 新人が検証しやすい理由
- 追加インストールほぼゼロ（Node 26 + `node:sqlite` か better-sqlite3 のみ）。
- 出力の差が目で見えるので、成功/失敗の判定が明快。

#### 想定読者
- UUIDの主キー選定で迷っている新人〜2年目
- 「v7が良い」と聞いたが自分で確かめたい人

#### タイトル案
1. Node 26の`randomUUIDv7()`と`randomUUID()`を書き比べて、並び順の違いを見た
2. UUIDv7とv4を実際に生成して、SQLiteの並びとtimestampで違いを確かめた
3. 「v7はソートできる」を鵜呑みにせず`randomUUIDv7()`を手元で試した

#### 検証でやること
- v7/v4を各1000件生成し、辞書順ソートして生成順と一致するか比較
- v7の先頭48bitからUnix時刻(ms)を復元して生成時刻と突き合わせ
- SQLiteに主キーとしてINSERT→`ORDER BY id`とrowid順の一致度を観察

#### 検証手順
1. `node -v`（26.1以上を確認）とプロジェクト初期化
2. v4/v7生成スクリプトを書き、配列をソートして順序比較
3. v7のtimestamp抽出関数を書いて誤差を確認
4. `node:sqlite`でテーブル作成→連番でINSERT→取得順を比較
5. 結果を表にまとめ、詰まった点（バージョン差・experimental扱い等）を記録

#### 記事構成案
1. はじめに / 2. なぜUUIDv7が話題か / 3. 事前に調べたこと（RFC 9562・追加バージョン） / 4. 環境構築 / 5. v4とv7を生成して比較 / 6. timestamp復元 / 7. SQLiteで並び順検証 / 8. 詰まった点 / 9. 分かったこと / 10. まとめ

#### 差別化ポイント
過去のNode 26記事（Temporal/ffi/text imports/Map.getOrInsert等）とは**別API**。かつ「利点の紹介」ではなく**自分で生成して並び・時刻・DB順を観測した一次ログ**にする。

#### 参考リンク
- Node.js v26.1.0 リリース（randomUUIDv7 追加）
- RFC 9562（UUIDv7仕様）
- `node:sqlite` ドキュメント

#### 注意点
- 追加バージョンは 26.1（26.0ではない）。冒頭でバージョンを明記。
- experimental/stable 状態を実行時の警告有無まで確認して書く。

### 3. CSS `sibling-index()`/`sibling-count()` でJSなしの段階アニメを作ってみた

#### 概要
リスト要素に `animation-delay: calc(sibling-index() * 60ms)` のように指定し、JSを一切使わずstaggered（時間差）アニメを作る。`sibling-count()` で総数依存のサイズ計算も試す。Playwright（channel: chrome）で描画を撮る。

#### なぜ今よさそうか

- Baseline未達＝**未対応ブラウザでどう見えるか**の切り分けが記事価値になる。

#### 新人が検証しやすい理由
- HTML+CSSだけ。calc()が分かれば書ける。
- 過去記事（anchor positioning / gap decorations / focusgroup）でCSS新機能をPlaywrightで撮る型が既にある。

#### 想定読者
- staggered animationをJSで書いていたフロント新人

#### タイトル案
1. `sibling-index()`でJSなしのstaggeredアニメを作ってみた

#### 検証でやること
- `sibling-index()`で遅延を段階付けしたフェードイン
- `sibling-count()`で総数に応じた幅/角度計算（放射メニュー等）

#### 検証手順

### 4. `Bun.cron()` の常駐スケジューラを外部cronなしで試す（簡易）

- **概要**: Bun 1.3.12 の `Bun.cron()` で数秒間隔ジョブを登録し、発火ログ・停止・多重登録の挙動を観察。外部cron/OSスケジューラ不要で完全ローカル。
- **タイトル案**: 「`Bun.cron()`で外部cronなしに定期実行を試した」「Bun単体でスケジューラが完結するのか手を動かした」

- **注意点**: 常駐プロセスなので検証は短時間タイマー＋明示終了で。experimental挙動をログで確認。

### 理由
