## 上位候補の詳細

### 1. JSなしでツールチップ/ポップオーバーをCSS Anchor Positioningで作ってみた

#### 概要
`anchor-name` / `position-anchor` / `anchor()` / `position-area` を使い、これまでFloating UIやPopper.jsでやっていた「ある要素に別要素をくっつける」配置をCSSだけで実装する。ツールチップ・ドロップダウン・エラー吹き出しを題材に、はみ出し回避（`@position-try` / フォールバック）まで触る。

#### なぜ今よさそうか
2025末〜2026にかけてAnchor Positioningを含むCSS新機能がBaselineに入りつつあり、国内でも「Baseline 2026」文脈で言及が増えている。一方、新人が実際にツールチップを作ってスクショで見せる「試してみた」記事はまだ少なく、解説記事が中心。

#### 新人が検証しやすい理由
HTMLとCSSだけで完結。ビルドもJSも不要。1ファイルのHTMLをブラウザで開くだけで動く。

#### 想定読者
Floating UI/Popper.jsやJSでの位置計算に苦労した経験のある新人フロントエンド。

#### タイトル案
1. 新人がCSS Anchor Positioningでツールチップを作ってみた（JS不要）
2. Floating UIをやめてCSSだけでポップオーバーを配置してみた
3. CSS Anchor Positioningを初めて触って詰まったところ（Safari非対応ほか）

#### 検証でやること
- ボタンにツールチップを紐付け（`anchor-name` / `position-anchor` / `anchor()`）
- `position-area` のグリッド配置と `anchor-center` を試す
- 画面端でのはみ出しを `@position-try` / フォールバックで回避
- `@supports` 付きフォールバックと、Playwright(Chromium)でのスクショ確認

#### 検証手順
1. 単一HTML＋インラインCSSで最小サンプルを作る
2. `anchor()` で上下左右に配置を切り替えて挙動を見る
3. `position-area` に置き換えて記述量を比較
4. ビューポート端に寄せてフォールバック動作を確認
5. Playwrightで各状態のスクリーンショットを撮り、対応/非対応ブラウザ差をメモ

#### 記事構成案
1. はじめに 2. なぜCSSだけで配置したいのか（JS実装の面倒さ） 3. 事前に調べたこと（Baseline状況） 4. 最小サンプル 5. 実際に試したこと（配置バリエーション） 6. 詰まった点（ブラウザ差・はみ出し） 7. 分かったこと 8. Floating UI比較 9. どんな人向きか 10. まとめ

#### 差別化ポイント
解説ではなく「実物をスクショで見せる検証ログ」。ブラウザ対応の詰まり所を新人目線で正直に書く。

#### 参考リンク
- MDN: CSS Anchor Positioning
- web.dev Baseline digest 2026 / css-tricks Interop 2026

#### 注意点
Safari/Firefoxは部分対応。「本番投入は`@supports`フォールバック前提」であることを明示する。Playwrightは基本Chromiumで確認する。

### 2. SQLiteしか知らない新人がTurso Database(Rust版)を触ってみた

#### 概要
SQLiteをRustで書き直したin-process DB「Turso Database」（0.6.1 / 2026-05）を`tursodb`シェルでローカルに動かし、SQLite互換の範囲・追加機能（ベクトル検索、並行書き込み/MVCC）を小さく試す。既存SQLiteとの差分と、まだBETAゆえの注意点を検証ログにする。

#### なぜ今よさそうか
「SQLiteのRust再実装」は海外で継続的に話題。libSQL/Turso含め動きが速く、日本語の実地検証ログは手薄。

#### 新人が検証しやすい理由
`curl`一発でCLI導入、アカウント不要・完全ローカル。SQLiteを知っていれば`.open`とSQLですぐ触れる。

#### 想定読者
SQLiteは使うが、Turso/libSQLの位置づけが分からない新人〜2年目。

#### タイトル案
1. SQLiteしか知らない新人がTurso Database(Rust版)を触ってみた
2. Rust製SQLite「Turso」をローカルで動かして素のSQLiteと比べてみた
3. Turso Databaseを試して詰まったところ（まだBETAの現実）

#### 検証でやること
- CLI導入と`tursodb`でのDB作成・基本CRUD
- SQLite DBファイルを開いて互換性を確認
- ベクトル検索など追加機能を最小で試す
- 素のSQLite(`sqlite3`)との体感差・非互換をメモ

#### 検証手順
1. インストールスクリプトでCLI導入
2. `tursodb`でin-memory→`.open file`で永続化
3. テーブル作成・INSERT/SELECT
4. SQLiteで作ったDBを開けるか確認
5. 追加機能を1つ試し、詰まり所と所要時間を記録

#### 記事構成案
1. はじめに 2. なぜTursoを試すか 3. libSQL/Tursoの整理 4. 導入 5. 触ってみた 6. 詰まった点（BETA/非互換） 7. 分かったこと 8. 素のSQLite比較 9. 向く人 10. まとめ

#### 差別化ポイント
libSQLとTurso Databaseの違いを新人目線で整理し、実際の互換性を手で確かめる。

#### 参考リンク
- GitHub tursodatabase/turso / Turso公式 local-first ページ

#### 注意点
BETA（本番非推奨）を明示。クラウド同期(Turso Cloud)はアカウントが要るため**ローカル完結部分だけ**を対象にする。
