# 出力テンプレート（検証ログ＝記事の素材）

`run-practice` の Step 6〜Step 8 で使う。実行して得た一次情報を、Zenn記事にそのまま
流し込める形で `logs/run-<slug>-<日時>/execution-log.md` に保存する。

> **境界**: このファイルは「記事の素材」まで。記事本文（articles/*.md）はここでは書かない。
> 「記事への写像」は、どの記録・どのスクショをどの見出しに使うかを**指し示す**ところまで。

## Output Format

最終出力は次のMarkdownテンプレートに沿って構成する。プレースホルダは実行結果で埋める。
チェックボックスは実行の実績（達成/未達）で更新する。エラーは全文を貼る。

```md
# 検証ログ: <テーマ名>

> Zenn記事の素材。実行の一次情報（コマンド・エラー全文・所要時間・スクショ）を記録。
> 記事本文は未執筆。「記事への写像」を見て articles/*.md に展開する。

## Image Handling（スクショの扱い / 記事化の申し送り）

* 素材段階では `logs/run-<slug>-<日時>/screenshots/` に置いたままでよい。
* **記事化する工程**では、Zenn の画像規約に合わせて `images/<slug>/` に移し、
  本文から `![説明](/images/<slug>/01-top.png)` の形で参照する
  （Zenn はリポジトリ直下の `/images` を配信する）。この移動は記事化担当が行う（本Skillはしない）。
* execution-log.md には「どのスクショをどの見出しで使うか」を書いておき、記事化を助ける。

## Slug Rules（保存ディレクトリ名）

`logs/run-<スラッグ>-<日時>/` のスラッグは対象 `practice-<スラッグ>-*.md` から引き継ぐ。
引き継げない場合のみ対象技術から作る（英語技術名を小文字ハイフン化。無理なら `task`）。
