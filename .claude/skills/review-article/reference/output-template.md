# レビューレポート テンプレート＆判定ルール

`review-article` の Step 5〜Step 7 で使う。公開可否の判定基準と、レポートの保存形式。

## 判定ルール（公開可否）

| 判定 | 条件 | 意味 |
|---|---|---|
| **公開不可** | blocker が1件以上 | 公開してはいけない。直してから再レビュー |
| **要修正** | blocker 0・warning が1件以上 | 公開前に直すべき点がある |
| **公開可** | blocker 0・warning 0（suggestion は可） | `published: true` にして push してよい状態 |

* 判定は必ず根拠（該当する指摘）とともに示す。
* 公開安全（published/秘密情報/slug）に関わる指摘は blocker に倒す。
* 出典ログが特定できず事実整合を確認できない場合は、最低でも **要修正**（warning）にする。

## Output Format

レビューレポートは次のMarkdownテンプレートに沿って構成し、`logs/review-<slug>-<日時>.md`
に保存する。

```md

- 修正の適用: なし（レポートのみ） / あり（下記「適用した修正」）
- 公開基準: 標準 / 厳しめ / ゆるめ

## 判定
```

## Slug Rules（レポートのファイル名）

`logs/review-<スラッグ>-<日時>.md` のスラッグは対象記事のファイル名（`articles/<slug>.md`）を
そのまま使う。日時は `date +%Y%m%d-%H%M`。

## 書き方の注意

* 指摘は「箇所（節/行）＋何を何に直すか＋根拠」の3点をそろえる。コピペで直せる粒度にする。
* 機械チェックの false positive は目視で切り分け、重大度を調整した理由を書く。
* レポートは非破壊レビューの記録。記事本文は（修正適用を指定されない限り）変更しない。
