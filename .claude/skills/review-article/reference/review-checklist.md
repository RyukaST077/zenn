# 公開前レビュー チェックリスト（観点）

`review-article` の Step 3〜Step 4 で使う。各次元の確認項目・重大度・確認方法。
機械チェック（`scripts/check-article.sh`）で一次検出できるものは (script) を付す。
最終判断は観点チェックで行う。

## 重大度の定義

* **blocker**: これがあると公開してはいけない。公開安全・事実の重大な誤り・秘密情報。
* **warning**: 公開前に直すべき。品質・体裁・軽微な事実の穴。
* **suggestion**: 直すとより良い。任意。

## 1. 公開安全（最優先）

| 項目 | 重大度 | 確認方法 |
|---|---|---|
| `published: false` になっている (script) | true なら **blocker** | Front Matter の published |
| 秘密情報が含まれない（APIキー/トークン/パスワード/秘密鍵/接続文字列） (script) | 含むなら **blocker** | 秘密情報パターンの grep ＋目視。プロンプト/ログ引用も見る |
| 内部情報・個人情報が無い（社内ホスト名、`/Users/<氏名>/...` 等の個人パス） (script) | 含むなら **blocker/warning** | 個人パス・内部URLの目視。公開して問題ないか |
| slug が妥当（文字種 `a-z0-9_-`・長さ12〜50） (script) | 違反は **blocker** | ファイル名 |
| slug が具体的で一意（汎用回避・ローカル重複なし） (script) | 汎用/重複は **blocker/warning** | `getting-started-*` 等の汎用語、`articles/` 内重複。knowledge 参照 |

## 2. Front Matter 妥当性

| 項目 | 重大度 | 確認方法 |
|---|---|---|
| 必須フィールド（title/emoji/type/topics/published）が揃う (script) | 欠落は **warning** | Front Matter |
| `type` が `tech` または `idea` (script) | 不正は **warning** | |
| `topics` が最大5個・英小文字で妥当 (script) | 超過/不正は **warning** | |
| `emoji` が絵文字1つ | 不正は **suggestion** | |
| `title` が誇大でない・長すぎない (script) | 誇大/長大は **warning/suggestion** | 「完全理解/徹底解説/保存版」等、長さの目安60字 |

## 3. 事実性（Fidelity / 出典ログとの整合）★核

出典ログ `execution-log.md` を正として照合する。

| 項目 | 重大度 | 確認方法 |
|---|---|---|
| 記事の結論（達成/一部/未達）がログの結果サマリーと一致 | 不一致は **blocker** | ログの「結果サマリー」と本文まとめを突合 |
| 実行コマンド・エラー全文がログに存在する（創作でない） | 創作は **blocker** | ログのタスク実行ログと本文コードブロックを突合 |
| 数値・所要時間がログに裏付けられる | 裏付け無しは **warning** | ログの実測値と本文の数値 |
| ログに無い成功・断定を書いていない | ある場合 **blocker/warning** | 「動いた/速い/簡単」等の断定に根拠があるか |
| 貼っているコードが実際の workspace 由来 | 創作コードは **warning** | workspace の抜粋か |
| `要素材` マーカーが残っていない (script) | 残存は **warning（未完成）** | `要素材` の grep |
| 出典ログが特定できている | 不明なら **warning** | 記事冒頭コメント/最新ログ |

## 4. 画像

| 項目 | 重大度 | 確認方法 |
|---|---|---|
| 本文の `/images/<slug>/...` 参照がすべて実在 (script) | 欠落は **blocker** | 参照パスを repo 直下で解決 |
| 画像に説明（alt テキスト）がある | 無しは **suggestion** | `![alt](...)` の alt |
| 完了条件・詰まった点にスクショが添えられている | 無しは **warning** | 本編/詰まった点の見出し |
| `images/<slug>/` に未参照の孤立画像が無い (script) | あれば **suggestion** | ディレクトリと参照の差分 |

## 5. Markdown 構造

| 項目 | 重大度 | 確認方法 |
|---|---|---|
| コードブロックの ``` が閉じている (script) | 未閉じは **blocker** | ``` 行数が偶数か |
| `:::message` / `:::details` が閉じている (script) | 未閉じは **warning** | `:::` 行数が偶数か |
| 見出し階層が破綻していない（H1 乱用なし・順序） | 破綻は **warning** | `#` の使い方 |
| リンクが壊れていない（`example.com`/`TODO`/空リンク無し） (script) | プレースホルダは **warning** | リンク先の目視 |
| 公式ドキュメントへの参考リンクがある | 無しは **suggestion** | 参考リンク節 |

## 6. 文章品質・トーン

| 項目 | 重大度 | 確認方法 |
|---|---|---|
| 経験談トーン（新人が試した範囲・断定しすぎない） | 逸脱は **suggestion** | 全体の語り口 |
| うまくいった点だけでなく詰まった点が書かれている | 無しは **warning** | 「詰まった点」節の有無・具体性 |
| 再現性（OS・ランタイム・バージョン）が明記されている | 無しは **warning** | 環境節 |
| 誤字・表記ゆれ・冗長がない | あれば **suggestion** | 目視 |
| 読者が最初に得る情報（結論・対象読者）が冒頭にある | 無しは **suggestion** | はじめに節 |

## 7. 完成度

| 項目 | 重大度 | 確認方法 |
|---|---|---|
| プレースホルダ（`<slug>`, `<...>`, TODO/FIXME）が残っていない (script) | 残存は **warning** | grep |
| 前提コメント `<!-- 前提: ... -->` が意図的（消し忘れでない） | 消し忘れは **suggestion** | 冒頭コメント |
| 記事全体として公開に耐える長さ・まとまり | 不足は **suggestion** | 全体量・構成 |

## レビューの姿勢

* まず機械チェックを走らせ、客観的に取れる指摘を確定する。
* 事実性（出典ログ照合）を丁寧に見る。ここが本パイプラインの記事の信頼性の要。
* 指摘は必ず「どこ（行/節）を・何に直すか」まで書く。抽象的な講評で終わらせない。
* false positive（例: 散文中の「password」という語）は、目視で切り分けて重大度を下げる。
* 迷ったら安全側（公開安全に関わるものは重めに）に倒す。
