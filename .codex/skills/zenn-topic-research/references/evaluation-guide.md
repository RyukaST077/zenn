# Evaluation Guide

## Candidate Selection Rules

優先するテーマ:

- 最近話題になっている
- 公式ドキュメントやチュートリアルがある
- 無料または低コストで試せる
- ローカル環境または無料枠で検証できる
- 半日から1日程度で最小検証できる
- 新人エンジニア目線で学びが出せる
- Zenn読者が興味を持ちやすい
- 既存記事と差別化できる
- 小さな成果物を作れる
- 詰まった点も記事の価値にできる

避けるテーマ:

- 大規模な本番環境が必要
- 高額なクラウド費用が発生しやすい
- 専門知識が重すぎる
- 検証に数日以上かかる
- 公式情報が少なすぎる
- セキュリティリスクが高い
- 規約やライセンス面が不明確
- 単なるニュース紹介で終わりそう
- 新人が実践するには難しすぎる

## Evaluation Criteria

各テーマ候補を5段階で評価する。

### Trend Strength

- 5: 直近で非常に話題になっている
- 4: 複数の技術コミュニティで注目されている
- 3: 一定の注目がある
- 2: ややニッチ
- 1: 話題性が弱い

### Beginner Feasibility

- 5: 公式チュートリアルだけで試せる
- 4: 少し調べれば試せる
- 3: 基礎知識があれば試せる
- 2: ある程度の実務経験が必要
- 1: 新人には重すぎる

### Article Potential

- 5: 検証ログとして非常に書きやすい
- 4: タイトルと構成が作りやすい
- 3: 工夫すれば記事にできる
- 2: 内容が薄くなりやすい
- 1: 記事化しづらい

### Practical Value

- 5: 実務や個人開発にすぐ役立つ
- 4: キャッチアップ価値が高い
- 3: 学習価値がある
- 2: 興味本位に近い
- 1: 実用性が弱い

### Verification Cost

- 5: 1から2時間で試せる
- 4: 半日で試せる
- 3: 1日あれば試せる
- 2: 数日かかる
- 1: 検証が重すぎる

### Differentiation

- 5: 新人目線、比較、失敗ログで差別化しやすい
- 4: 検証内容を工夫すれば差別化できる
- 3: 多少の差別化は可能
- 2: 既存記事と似やすい
- 1: 焼き直しになりやすい

## Topic Conversion

技術名をそのまま出さず、記事として書ける形に変換する。

Bad:

- Bun
- Cloudflare Workers
- AIエージェント
- Drizzle ORM
- Playwright
- Supabase

Good:

- Bunを初めて触って、Node.jsとの違いを小さなAPIで試してみた
- Cloudflare Workersで新人が初めてサーバーレスAPIを作ってみた
- AIコーディングツールを使ってTodoアプリの実装をどこまで任せられるか試してみた
- Prismaしか知らない新人がDrizzle ORMを触ってみた
- Playwrightで初めてE2Eテストを書いてみた
- Supabase AuthをNext.jsアプリに組み込んでみた

## Title Patterns

### Tried It

- `新人エンジニアが〇〇を触ってみた`
- `話題の〇〇を1日試してみた`
- `〇〇を初めて使って△△を作ってみた`
- `〇〇の公式チュートリアルをやってみた`

### Beginner Perspective

- `〇〇を初めて触って分かったこと`
- `〇〇初心者が最初に詰まったポイント`
- `〇〇を新人目線で整理してみた`
- `〇〇が分からなかったので小さく試してみた`

### Comparison

- `〇〇と△△を新人目線で比べてみた`
- `〇〇しか知らなかった新人が△△を試してみた`
- `〇〇と△△、最初に触るならどちらがよさそうか`

### Small App

- `〇〇で小さなTodoアプリを作ってみた`
- `〇〇をNext.jsアプリに組み込んでみた`
- `〇〇を使って簡単なAPIを作ってみた`
- `〇〇でミニアプリを作って理解する`

### Failure Log

- `〇〇を試して詰まったところと解決方法`
- `〇〇の環境構築でハマったこと`
- `〇〇を動かすまでにやったこと`
- `〇〇でエラーが出たので原因を調べた`

## Anti-Patterns

- 実践せずにニュースをまとめるだけの記事
- 公式ドキュメントの要約だけの記事
- 新人には検証できない高度すぎるテーマ
- 高額なクラウド費用がかかるテーマ
- 環境構築だけで終わるテーマ
- 話題性の根拠がないテーマ
- `完全理解` や `徹底解説` など、経験値に対して強すぎるタイトル
- 調査範囲が広すぎるテーマ
- 1記事で複数の技術を詰め込みすぎるテーマ
- 海外で話題だが日本語読者への価値が薄いテーマ
