# 実践タスク: Node 26 の `Map.getOrInsert` と `Iterator.concat` で「get-or-set定型」を書き比べてみた

## このタスクの前提

- 出典レポート: `research/search-topic-20260716-0400.md`

- 対象技術: Node.js 26（同梱 V8 14.6）の新コレクションメソッド `Map.prototype.getOrInsert` / `getOrInsertComputed`（WeakMap版含む）と `Iterator.concat`

- 想定筆者 / 想定読者: Web系の新人エンジニア / 新人〜実務2年目（Map・配列でグルーピングや集計を書いたことがある層）

- 実行環境の担保: `node` 一発で完結（追加依存ゼロ、認証・課金・ネットワーク不要）。数値比較の可視化が必要な箇所のみ、生成した結果表をHTML化して **Playwright でスクショ**する。すべてAIエージェント単独（CLI / コード / Playwright）で実行・検証可能。テーマの置き換えは不要。

> **裏取り済みの前提（一次情報）**
> - Node.js **26.0.0**（2026-05-05 リリース）が **V8 14.6.202.33** を同梱。対象APIは**フラグ不要・既定で有効**。（出典: nodejsdesignpatterns.com「What's new in Node.js 26」/ nodejs.org リリースノート）
> - `Map.prototype.getOrInsert(key, defaultValue)` … キーがあれば取得、なければ `defaultValue` を入れて返す。MDN上は **Baseline 2026（Newly available, 2026-02）**。
> - `Map.prototype.getOrInsertComputed(key, callbackFn)` … コールバックは**キー未存在時のみ**実行。**戻り値がそのまま格納**される（Promiseを返すとPromiseが格納される点に注意）。
> - 両メソッドは `WeakMap` にも同シグネチャで存在。
> - `Iterator.concat(...iterables)` … 複数イテラブルを中間配列なしで遅延連結する **静的メソッド**。**同期専用**（async iterableは対象外）。
> - ⚠ **要確認/最大の注意**: この環境の `node -v` は **v22.17.0**。対象APIは Node 26 でしか動かないため、**まず Node 26 の実行環境を用意する**ことが本タスクの最初の関門（フェーズ2）。

## 詰まりそうなポイントと対処の指針

| # | 詰まりそうな点 | なぜ起きるか | 最初に試すこと | 記事での活かし方 |
|---|---|---|---|---|
| 1 | `map.getOrInsert is not a function` / `Iterator.concat is not a function` | 実行中の Node が26未満（現環境は v22.17.0）。対象APIは Node 26 / V8 14.6 でのみ有効 | `node -v` を確認し `nvm use 26`。シェルを開き直した際に旧バージョンへ戻っていないかも確認 | 「まずランタイムの壁」を冒頭に。動かした `node -v` と日付を明記する重要性を実体験として書く |
| 2 | `getOrInsertComputed` で重い処理が毎回走る/1回も走らない、と勘違い | `getOrInsert(k, expensive())` は第2引数を**即時評価**、`getOrInsertComputed(k, fn)` は**未存在時のみ** `fn` 実行、という違い | 呼び出し回数カウンタを仕込んで実際の実行回数をログ化（フェーズ3のタスク） | 「`getOrInsert` と `getOrInsertComputed` の使い分け」を実測ログ付きで解説（記事の核） |
| 3 | `getOrInsertComputed(k, async () => ...)` の値が期待と違う | コールバックの**戻り値がそのまま格納**される。async関数を渡すと Promise が Map に入る | 同期関数を渡す。非同期が必要なら別途 await してから `set`／`getOrInsert` する | 「地味だがハマる」落とし穴として1節。公式の注意書きへリンク |
| 4 | `Iterator.concat` に配列を渡してうまくいかない/async で使えない | `Iterator.concat` は同期イテラブル前提の静的メソッド。async iterable は対象外 | まずジェネレータ/同期イテレータで試す。async は自前 `async function*` で連結 | 「できること/できないこと」の線引きを明示。過度な期待を戒める経験談に |
| 5 | ベンチの数値がブレて「速い」と言い切れない | JIT ウォームアップ・GCタイミング・単一環境での計測誤差 | 試行回数を増やし中央値を見る、`--expose-gc` で計測前にGC、断定を避ける | 「簡易計測なので参考値」と正直に書くことで信頼性を上げる（レポートの注意点に合致） |
