### 裏取りで確認した一次情報（2026-07-02 時点）

- Vite 8 は **2026-03-12 に安定版リリース**。Rolldown（Rust製バンドラ）が**デフォルト統合**（フラグ不要）。（Vite公式blog: announcing-vite8）
- 必要ランタイム: **Node.js 20.19+ または 22.12+**。（同上）
- インストールサイズは約15MB増（lightningcss ~10MB / Rolldown ~5MB）。多くのプロジェクトは設定変更なしで移行可能とされるが「複雑なプロジェクトは段階的移行推奨」。（同上）
- Vite 7 に居ながら Rolldown を先行検証するには `rolldown-vite`（一時パッケージ）を package.json の overrides で差し込む。（v7 docs: guide/rolldown）
  - npm: `"overrides": { "vite": "npm:rolldown-vite@latest" }`
  - pnpm: `"pnpm": { "overrides": { "vite": "npm:rolldown-vite@latest" } }`
  - 既知警告: `Warning validate output options ... Invalid key` はRolldown未対応のRollupオプションが原因。実験的につきバージョン固定推奨。
- Rolldown 1.0 は 2026-05 安定版。ただし「出力挙動（DCE・チャンク戦略）は今後も改善余地あり」との注記。（voidzero: announcing-rolldown-1-0）

## 完成イメージ（成果物）

- 作るもの: **同一の最小Reactアプリ（TS）を Vite 7 と Vite 8 の2構成で用意し、build/dev起動時間を同条件で複数回計測した比較表**（＋移行時のつまずきログ＋起動画面スクショ）
- 「できた」と言える完了条件:
  1. Vite 7 で `build` と `dev` 起動が成功し、cold/warm を各3回計測できている
  2. Vite 8 に移行後、同じアプリで同条件の計測が取れている
  3. 両者の数値を1つの比較表にまとめ、差（倍率）を出せている
  4. `vite preview`（または dev）で表示した画面を Playwright でスクショ保存できている
- 完了確認の方法: `time`（またはビルドログ出力の所要時間）＋ 比較表Markdown ＋ Playwrightスクショ（`localhost` 表示）
- 記事タイトル案（そのまま使える形）:
  1. 新人がVite 8（Rolldown）に移行して、ビルド時間を実測してみた
  2. 「10〜30x速い」は本当か？ Vite 7→8を手元で計ってみた
  3. Vite 8移行で詰まったところと、実際のビルド時間の変化

## 想定リスク・注意点

- コスト: 完全無料・ローカル完結。課金トリガーなし。
