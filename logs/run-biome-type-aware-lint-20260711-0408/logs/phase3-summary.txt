学び:
1) 今回の3バグは全て nursery ルール。types:"recommended" では拾えない。
2) types:"all" にしても自分の環境では自動発火せず、rules.nursery で各ルールを
   "error" 明示して初めて3つとも発火した（詰まりポイント#1が的中）。
3) 型認識ルールを個別に有効化すると、domains.types を "none" にしても検出は消えない。
   → ルールの有効化そのものが Biome Scanner（型推論エンジン）を起動する。
   → 「型認識ON/OFFの対比」は domains トグルではなく、ルール自体のON/OFF（=素の
     recommendedとの比較）で見るのが正確。
4) 型認識ONのときだけ --verbose に "Scanned project folder" が出る。実行時間も
   素recommended(約250ms) と scanあり(約950msまで振れる) で体感差があった。
5) noUnsafePlusOperands は number+string を「文字列結合」として許容し、
   number+bigint 等の混在だけを不正とする。typescript-eslintのrestrict-plus-operandsと
   検出範囲が違う点に注意（最初 number+string で書いて発火せず気づいた）。
